# CHANGE CONTROL: Modification of this list requires RFC and IT Architecture approval.
#
# Opposite direction of Lists\ApprovedRemoval-Capabilities.txt: these are
# Features on Demand to ADD to the image (offline), not remove. Requires a
# local FOD source - see Scripts\14-Add-FeaturesOnDemand.ps1 -FodSourcePath.
# Capability names are versioned and can drift between Windows builds - like
# Lists\ApprovedRemoval-Capabilities.txt, entries here are resolved against
# the mounted image with prefix matching (up to the version tilde) before
# being passed to Add-WindowsCapability, so you don't have to keep this file
# in lockstep with every Windows build's exact version suffix.
