
# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 2.x     | Yes       |
| 1.x     | No        |

## Reporting a Vulnerability

Please do not publicly disclose security vulnerabilities.

Submit reports privately with:

* Description
* Impact
* Reproduction steps
* Suggested mitigation

Reports should include sufficient technical detail to validate and reproduce the issue.

## Scope

Examples include:

* Privilege escalation
* Credential exposure
* Unsafe unattended configuration
* Weak security baseline implementation
* Supply-chain risks
* Script execution vulnerabilities
* Insecure defaults

## Out of Scope

* Cosmetic issues
* Documentation typos
* Feature requests
* Unsupported operating systems

Security reports will be reviewed and addressed according to severity and impact.
